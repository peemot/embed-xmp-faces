# XMP Face Embedder

[![Python 3.x](https://img.shields.io/badge/Python-3.x-blue.svg)](https://www.python.org/downloads/)

A fast, bulk-processing Python script that extracts face metadata and subject tags from `.xmp` sidecar files and embeds them directly into the corresponding image files using `exiftool`. 

Perfect for making face recognition data (generated by tools like Immich, digiKam, or Lightroom) portable by baking it directly into your `.jpg`, `.png`, or `.tiff` files.

## Features
* **Lightning Fast:** Uses a "two-pass" batch execution method to evaluate and modify thousands of images in seconds.
* **Surgical Precision:** Only writes `<mwg-rs:Regions>` (Face bounding boxes) and `<dc:subject>` (Tags/Names). Ignores other metadata from the sidecar.
* **Unknown Filter:** Automatically filters out unassigned face boxes ("Unknown") to keep your photo metadata perfectly clean.
* **Auto-Population:** Automatically creates standard Subject tags for any named faces if they are missing from the XMP.
* **Smart Merging:** Can safely append new faces to images that already have existing metadata without creating duplicate bounding boxes.
* **Detailed Logging:** Generates clean, timestamped logs of modified and skipped files.

## Prerequisites
1. **Python 3.6+**
2. **[ExifTool](https://exiftool.org/):** Must be installed and accessible via your system's `PATH` variable.

## File Structure Requirement
The script looks for matching sidecar files based on the exact original filename with `.xmp` appended to the end:
```text
📁 photos/
 ├── 📄 vacation.jpg
 ├── 📄 vacation.jpg.xmp   <-- (Correct)
 ├── 📄 family.jpeg
 └── 📄 family.jpeg.xmp    <-- (Correct)
```

## Usage

Run the script from your terminal, pointing it to your root pictures directory:

```bash
python embed_faces.py /path/to/my/photos
```

### Command-Line Arguments

| Flag | Description |
| :--- | :--- |
| **(default)** | Skips files that already contain face/subject metadata. Ignores "Unknown" faces. Overwrites the original image directly without leaving a backup. |
| `--merge` | Do not skip images with existing metadata. Instead, carefully append the new XMP faces and tags without creating exact duplicates. |
| `--include-unknown` | Include bounding boxes and subject tags for faces explicitly named "Unknown" (by default, these are safely dropped). |
| `--keep-original` | Tells ExifTool to keep the original, unmodified image file as a backup (e.g., renames it to `vacation.jpg_original`). |
| `--debug` | Generates a secondary, highly detailed timestamped log file containing raw JSON structures and ExifTool execution arguments. Great for troubleshooting weirdly formatted XMP files. |

## Examples

**Merge new XMP data with existing photo tags, keeping original backups:**
```bash
python embed_faces.py /path/to/photos --merge --keep-original
```

**Include "Unknown" unassigned faces when embedding:**
```bash
python embed_faces.py /path/to/photos --include-unknown
```

**Run in debug mode to inspect why a specific folder's tags aren't applying:**
```bash
python embed_faces.py /path/to/problem_folder --debug
```
